---

## 1. 执行摘要

我们正在为有家庭血压监测需求的用户（父母自录 + 子女辅助查看）构建一款微信小程序，以日历打卡记录为核心、辅以血压/心率趋势折线图，帮助用户建立规律的测量习惯并直观理解血压变化趋势。当前痛点在于：靠记忆或纸笔记录既不持续、也难以发现长期规律。

---

## 2. 问题陈述

### 谁有这个痛点？
- 50–60 岁中老年用户（父母），手机熟练度一般，需要自行录入血压数据
- 20–30 岁子女，关心父母健康、辅助查看趋势

### 痛点是什么？
家庭血压测量后缺乏便捷、持续、可视化的记录工具。纸笔记录容易中断且无法分析趋势，现有健康 App 往往功能冗余、入口深、字号小，对中老年不友好。

### 为什么重要？
- **用户维度**：不知道血压长期走向，难以在就医时提供连续数据
- **行为维度**：没有记录反馈闭环，测量动力不足，容易从"每周测"退化为"偶尔测"

---

## 3. 目标用户与画像

### 主要用户：父母
- 年龄 50–60 岁，手机熟练度一般
- 偏好简洁直白的操作，不喜欢多层级菜单
- 需要大字视图，避免误触
- 测量频率目标：每天一次（晚间）

### 次要用户：子女
- 年龄 20–30 岁，手机熟练度高
- 辅助父母设置（范围、字号）
- 通过趋势图表查看父母血压变化

## 7. 用户故事与验收条件

### 🏠 首页

#### Story 1：快速录入血压记录

> 作为测量者，我想在首页一键进入录入页面，输入高压、低压、心率和用药情况，以便每次测完能快速完成记录。

**验收条件：**
- [ ] 首页有醒目的「+ 记录」按钮，点击进入录入页
- [ ] 录入页包含 4 个字段：收缩压（数字键盘）、舒张压（数字键盘）、心率（数字键盘）、备注/用药（文本）
- [ ] 提交时自动记录当前时间作为测量时间
- [ ] 字段校验：不为空、不为负数；心率范围 30–250
- [ ] 提交成功后返回首页，日历和数据自动刷新

---

#### Story 2：日历打卡总览

> 作为测量者，我想在首页看到一个日历视图，用颜色标记每天血压是否正常，以便一眼看出当月整体情况。

**验收条件：**
- [ ] 日历默认显示当月，支持左右箭头翻月
- [ ] 当日日期下方显示颜色圆点：
  - 🟢 绿色 — 当天所有记录均在正常范围内
  - 🔴 红色 — 当天至少有一条高于正常范围上限
  - 🟡 黄色 — 当天至少有一条低于正常范围下限（且无偏高）
- [ ] 未测量日期无标记
- [ ] 点击已测量日期，跳转当日详情

---

#### Story 3：最近记录预览

> 作为测量者，我想在日历下方看到最近三次记录摘要，以便快速回顾。

**验收条件：**
- [ ] 日历下方展示最近 3 条记录（时间倒序）
- [ ] 每条显示：日期、时间、收缩压/舒张压、心率
- [ ] 异常值旁有颜色标识
- [ ] 底部「查看全部记录」入口
- [ ] 无记录时显示空状态占位

---

### 📊 数据

#### Story 4：血压趋势折线图

> 作为子女，我想看到血压变化的折线图（含 120/80 参考线），以便快速判断趋势。

**验收条件：**
- [ ] 双折线：收缩压（红线）、舒张压（蓝线）
- [ ] 两条恒定参考线：120 mmHg、80 mmHg（灰色虚线），不受自定义范围影响
- [ ] Y 轴 mmHg，X 轴日期；同日多条取均值
- [ ] 默认最近 7 天，可切换 30 天
- [ ] 少于 2 条记录时显示空状态提示

---

#### Story 5：心率趋势折线图

> 作为子女，想同时追踪心率变化趋势。

**验收条件：**
- [ ] 独立折线图（绿色），位于血压图下方
- [ ] Y 轴次/分钟，X 轴日期；同日多条取均值
- [ ] 与血压图同步切换 7/30 天
- [ ] 少于 2 条记录时显示空状态提示

---

### 👤 我的

#### Story 6：自定义血压正常范围

> 作为使用者，想自定义正常范围阈值，以便根据自身情况设定标准。

**验收条件：**
- [ ] 默认值：收缩压 90–129，舒张压 60–89
- [ ] 提供滑块或数字输入设置上下限
- [ ] 修改后日历颜色实时生效
- [ ] 图表 120/80 参考线恒定不变
- [ ] 提供「恢复默认」按钮

---

#### Story 7：调整字号

> 作为年长使用者，想放大字体以便看清内容。

**验收条件：**
- [ ] 三档可选：小 / 中（默认） / 大
- [ ] 切换后全局生效

## 11. 技术约束与依赖

### 微信小程序硬约束

1. **框架限定**：仅使用 WXML / WXSS / JavaScript / JSON 原生技术栈。禁止 React、Vue、Angular；禁止 DOM/BOM API
2. **目录规范**：业务代码根目录 `miniprogram/`，严格遵循 `pages/` / `components/` / `app.js` / `app.json` / `app.wxss` 结构
3. **API 与组件**：所有网络/存储/UI 必须使用 `wx.*` 原生 API 和原生组件
4. **测试验收**：UI 交互以微信开发者工具模拟环境为准，不套用浏览器端 E2E 工具
5. **图表渲染**：使用微信 Canvas 2D API 绘制折线图，不依赖第三方图表库

### 数据存储

- 血压记录使用 `wx.setStorageSync` 本地存储
- 数据结构：`{ id, systolic, diastolic, heartRate, medication, timestamp }`
